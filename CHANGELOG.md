## [1.2.0] — Sprint 4 — Подписка и модерация

### Added
- Публикация рецептов пользователями с подпиской (`role=SUBSCRIBER`)
- Очередь модерации (`isModerated=false`) для пользователей с ролью `MODERATOR`
- Комментарии модератора при отклонении рецепта
- Уведомления автора об изменении статуса рецепта
- Интеграция платёжной системы (базовый сценарий)

### Changed
- Расширена модель `User`: добавлены поля `subscription_expires_at`, `role`
- Обновлён BPMN-процесс публикации в `/docs`

---

## [1.1.0] — Sprint 3 — Личный кабинет

### Added
- Личный кабинет: избранные рецепты (`Favorites`)
- Планировщик питания (`MealPlan`) — создание плана на неделю
- Автогенерация списка покупок (`generateShoppingList()`)
- Страница «Мои рецепты»

### Fixed
- Некорректный расчёт КБЖУ при масштабировании порций менее 0.5

---

## [1.0.1] — Sprint 2 (hotfix)

### Fixed
- Ошибка 500 при фильтрации по нескольким аллергенам одновременно
- Отображение нулевых значений КБЖУ для рецептов без указанного состава

---

## [1.0.0] — Sprint 2 — Фильтрация и КБЖУ

### Added
- Фильтрация рецептов по диапазонам КБЖУ (калории, белки, жиры, углеводы)
- Исключение аллергенов (`User_Exclusions`)
- Масштабирование порций (`scaleIngredients()`)
- Предустановленные диеты: «Без глютена», «Веган», «Кето»

---

## [0.1.0] — Sprint 1 — MVP Core

### Added
- Регистрация и авторизация пользователей (JWT)
- Поиск рецептов по ингредиентам (режимы: «все» / «хотя бы один»)
- Карточка рецепта: фото, КБЖУ, время, сложность, рейтинг
- Базовая БД: таблицы `Users`, `Recipes`, `Ingredients`, `Recipe_Composition`
- Docker Compose: Backend + PostgreSQL + Nginx
- CI/CD: GitHub Actions (pytest, Jest, pylint, ESLint)
